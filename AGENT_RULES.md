# Agent Rules

These rules apply to AI agents working inside the Orinadus repository.

Read `PROJECT_CONTEXT.md` and `ORINADUS_CONTEXT.md` before making architectural changes.

## Primary Rule

Treat Orinadus as an intelligence and authored-interpretation system.

Do not collapse it into a generic chatbot, orchestration harness, or catch-all publishing bucket.

## Scope Discipline

Changes in this repository should strengthen one or more of the following:

- source discovery
- signal handling
- knowledge organization
- author desks
- authored interpretation
- report and briefing workflows

## Ecosystem Boundary Rules

- orchestration infrastructure belongs more naturally to `Autiladus`
- persistent companion behavior belongs more naturally to `NQITA`
- creator tooling belongs more naturally to `WokStudio`

Integration is acceptable; ownership drift is not.
